01 — Foundations

Principles

Four rules that keep AUROR feeling like AUROR. Apply them when extending tools, panels, exports, or anything new.

01
Geometry first

Pixels serve the drawing. Chrome stays calm so the canvas stays loud. Avoid decoration that competes with strokes.

02
Lightweight, like the engine

Single file, no install, no plugins. The UI matches: small surfaces, low file weight, fast paints, minimal motion.

03
Hackable defaults

Tokens, JSON projects, plain CSS. Every choice should be readable at a glance and easy to fork.

04
Small mark energy

Friendly, never cute. Italian-first, English-equal. A wink in the copy, never in the chrome.

05 — Shape

Radii & elevation

Generously rounded but never blobby. Shadows are soft and warm, never theatrical — they hint at elevation without faking glass.

Radii

Use: One radius — 10 px — for everything. Buttons, inputs, cards, panels, modals, badges, tags, the language select, the layer chip. No pills. No mixed corners. The colour swatch keeps its 50% circle because it represents a colour token, not a UI shape.

06 — Motion

Motion is optional

Motion confirms an action — it never narrates one. Three durations, one curve. If a transition longer than 220 ms feels right, it's probably wrong for AUROR.

FAST
--auror-dur-fast
80 ms · transform on press, dock active flash
BASE
--auror-dur-base
120 ms · button bg, tab swap, hover state
SLOW
--auror-dur-slow
220 ms · panel reveal, modal fade
cubic-bezier(.2,.7,.2,1)
--auror-ease
A single ease for the whole app.

Don'ts: No bounces, no springs, no parallax. Reduced-motion users get instant transitions — wrap any new animation in @media (prefers-reduced-motion).

08 — Components

Components

The building blocks that ship today, plus the spec for how they should behave when extended.

Button

36–40 px tall · radius 10 · border 1 px line · gap 8 between icon + label

Hierarchy: One .primary per visible card. Default buttons are neutral; danger is reserved for irreversible actions. Stati binari (Snap, Ortho, Griglia) usano lo .aurorSwitch, mai un button. Icon + label in italiano nelle barre principali; icon-only solo nel dock e nei menu di contesto.

Tool dock

Vertical, fixed 64 px wide · 44 px tools · 10 px gap

Anatomy

  • 1
    Container. Rounded card, soft shadow, 8 px horizontal / 10 px vertical padding.
  • 2
    Tool button. 44×44, radius 10, 1 px border. Default fill #f0f2f5, hover lifts 1 px and gains a soft shadow, active gets the brand-soft fill + ring.
  • 3
    Group separator. 36 px wide hairline, 4 px above/below. Group: draw → edit → utility.
  • 4
    Tooltip on hover (delay 400 ms) shows the localized data-i18n-title string, never raw IDs.

Layer list

One row per layer · 28 px iconBtns · active row gets a brand-soft button background

Notes

  • ·
    Click the colored dot opens the color picker. Don't try to inline a swatch grid here.
  • ·
    The active layer has a brand-soft background on the name button — never on the row itself, so visibility/lock controls stay neutral.
  • ·
    Hidden rows drop the name to 55 % opacity. Locked rows drop to 75 %. Both keep the dot at full saturation so layer identity is preserved.
  • ·
    Long names truncate with text-overflow: ellipsis rather than wrapping.

Canvas context menu

Right-click anywhere on the canvas · keep flat, no submenus

Rules

  • ·
    One flat list, max ~7 items. Group related actions with .ctxSep.
  • ·
    Every entry has an icon left-aligned in a 20×20 box, even if the icon is light.
  • ·
    Disabled items use opacity .45 — never red text — so they remain readable in context.

Color picker

A rectangular HSV panel — saturation/value square, hue strip, swatches, hex input

Why rectangular

The picker uses the same vocabulary as the rest of the chrome — a card with a 10 px radius, hairline borders, a hex input that matches every other .aurorInput, and the same Annulla/OK button pair the inspector and modals use. No bespoke shapes.

  • ·
    The SV square is the primary surface — drag for saturation (x) and value (y). Aspect ratio is locked 1:1 so it never distorts.
  • ·
    The hue strip sits directly below at 12 px tall — same height as a hairline divider plus a thumb.
  • ·
    Eight recent swatches persist per project. The active one gets a brand-ring outline, never a fill change.
  • ·
    The hex field is editable and the canonical value — typing #F59E0B wins over the SV/hue cursors.

10 — Voice

Voice & tone

Friendly, technical, slightly self-aware. AUROR knows it's a small CAD; it doesn't apologize for it. The README sets the bar — match its tone in product copy, never exceed it.

Do

  • ✓
    Be terse. “Pronto”. “Snap: on”. “Zoom: 5”. Statusbar lines never wrap.
  • ✓
    Speak in coordinates. When a number is the answer, lead with the number. x: 142.0 y: 88.5.
  • ✓
    Italiano e inglese. Default in IT, EN come lingua paritaria. Stesse lunghezze di stringa in entrambe — EN non deve mai essere troncato.
  • ✓
    Verbi all'imperativo. Salva, Carica, Applica, Elimina. Mai “Clicca qui per salvare”.

Don't

  • ✕
    No emoji nel chrome. Il brand vive nella documentazione, non nei pulsanti. L'app resta SVG puro.
  • ✕
    No tono marketing. “Unlock pro features”, “Potente, intuitivo…” — AUROR non parla così.
  • ✕
    Niente bottoni-frase. Un bottone è un verbo + (opzionale) un sostantivo. “Salva”, non “Salva il mio progetto”.
  • ✕
    Niente punti esclamativi in errori o stati vuoti. Tono calmo, tecnico, fattuale.
